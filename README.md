Данные playbookи нужны для подготовки серверов Ubuntu к объединению в кластер k8s. 

Перед началом выполнения playbookов необходимо сделать:
 - На каждом хосте настроить SSH сервер с возможностью подлключения с ansible master server.
 - Вписать в файл hosts.txt адреса серверов (в данном примере были созданы группы)
 - Настроить group_vars

Процесс разбит на 3 playbooka, которые надо произвести последовательно:
1. swap_off_and_reboot.yml
   - Проверка доступности хостов
   - Отключение swapa памяти (Так как кластер плохо работает с включённым свапом)
   - После произведённых настроек перезагружаем хост и дожидаемся, что он поднялся.
2.  install_docker.yml
   - Обновляем информацию о пакетах из репозитория
   - Устанавливаем необходимые нам пакеты
   - Добавляем Docker GPG key
   - Добавляем актуальный репозиторий Docker
   - Устанавливаем docker и запускаем службу
   - Добавляем текущего пользователя в группу docker и выводим информацию о версии на экран
3. install_k8s.yml
   - Создаём директорию /etc/apt/keyrings с определёнными правилами
   - Вписываем откуда будем брать gpg key и скачиваем его сюда /etc/apt/keyrings/kubernetes-apt-keyring.gpg
   - Ставим на /etc/apt/keyrings/kubernetes-apt-keyring.gpg определённые права и прописываем репозиторий k8s
   - Делаем обновление пакетов из репозитория
   - устанавливаем необходимые для работы и администрирования кластера пакеты - kubelet, kubeadm, kubectl
   - Фиксируем версии скачанных пакетов, что бы он не обновился


Теперь мы готовы настраивать Ноды и объединять их в кластер
